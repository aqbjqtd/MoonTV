# MoonTV 项目状态文档

## 📋 项目概览
**MoonTV** - 基于Next.js 14的现代化全栈影视聚合播放器，支持多源资源聚合、在线播放、用户数据同步等核心功能。

### 🎯 项目特色
- **多源聚合**: 20+视频资源站整合
- **智能搜索**: 去重、排序、质量评估
- **多播放器**: ArtPlayer + Vidstack 双重保障
- **PWA支持**: 离线可用，安装即用
- **多存储后端**: LocalStorage | Redis | Upstash | Cloudflare D1
- **响应式设计**: 完美适配桌面端和移动端

## 🏗️ 技术架构

### 核心技术栈
- **前端框架**: Next.js 14.2.23 (App Router + RSC)
- **开发语言**: TypeScript 4.9.5 (严格类型检查)
- **样式方案**: Tailwind CSS 3.4.17 + Headless UI 2.2.4
- **状态管理**: React 18 + Context API
- **包管理器**: pnpm 10.14.0 (性能优化首选)
- **构建工具**: Next.js + Webpack + SWC

### 视频播放技术
- **主播放器**: ArtPlayer 5.2.5 (功能丰富)
- **备用播放器**: Vidstack 0.6.15 + @vidstack/react 1.12.13
- **流媒体支持**: HLS.js 1.6.6 (M3U8流媒体)
- **编码处理**: he 1.2.0 (HTML实体编码)

### 数据存储架构
- **统一接口**: IStorage 抽象层，支持四种存储后端
- **数据类型**: 播放记录、收藏夹、搜索历史、用户认证、管理配置
- **存储策略**: 按用户名命名空间隔离，支持数据同步

## 📊 用户账号分享检测系统 - 详细规划

### 🎯 检测目标
- **主要目标**: 检测用户账号是否存在分享行为
- **次要目标**: 为精细化运营提供数据支持
- **限制条件**: 最小化对正常用户的影响

### 🔍 检测维度

#### 1. 地理位置分析
- **检测内容**: 用户登录地理位置变化
- **异常模式**: 短时间内不同城市/国家登录
- **风险评估**: 基于距离的动态评分

#### 2. 网络环境分析
- **检测内容**: IP地址、ISP、网络类型
- **异常模式**: 多个不同ISP的IP地址
- **风险评估**: 数据中心IP、VPN检测

#### 3. 设备特征分析
- **检测内容**: 设备指纹、浏览器信息
- **异常模式**: 不合理的设备组合
- **风险评估**: 设备多样性评分

#### 4. 行为模式分析
- **检测内容**: 播放时间、使用频率、并发情况
- **异常模式**: 24小时不间断使用、多设备并发播放
- **风险评估**: 行为模式偏离度

### 📊 风险评估模型

#### 风险等级划分
- **低风险 (0-30分)**: 正常使用行为
- **中风险 (31-60分)**: 需要关注的行为
- **高风险 (61-80分)**: 可能的分享行为
- **极高风险 (81-100分)**: 高度疑似分享

#### 自动化处理策略
- **低风险**: 记录日志，不采取行动
- **中风险**: 生成告警，管理员关注
- **高风险**: 管理员介入调查
- **极高风险**: 建议限制账号

### 🛡️ 用户体验保护

#### 误判预防
- **人工审核**: 所有高风险行为都需要人工确认
- **申诉机制**: 用户可以对判定结果提出申诉
- **白名单机制**: 特殊情况可以加入白名单

#### 隐私保护
- **数据匿名化**: 收集的数据进行匿名化处理
- **目的限制**: 数据仅用于账号分享检测
- **存储期限**: 数据定期清理，不过度存储

# Docker镜像优化计划

## 当前问题
- 镜像大小：1.53GB，过大
- 需要进一步分层优化以获得最小体积

## 优化策略

### 1. 多阶段构建优化
- **第1阶段(deps)**: 仅安装依赖，不复制源码
- **第2阶段(builder)**: 构建应用，清理不必要文件
- **第3阶段(runner)**: 最小运行时，仅复制必需文件

### 2. 依赖优化
- 使用 `--ignore-scripts` 避免执行不必要的脚本
- 使用 `--frozen-lockfile` 确保依赖一致性
- 在builder阶段清理开发依赖

### 3. 文件优化
- 仅复制运行时必需的文件
- 排除不必要的文件（.git, .vscode, node_modules中的开发依赖）
- 使用 .dockerignore 优化构建上下文

### 4. 基础镜像优化
- 使用 Alpine Linux 作为基础镜像
- 考虑使用 distroless 或 scratch 镜像进一步减小体积

### 5. 安全优化
- 使用非root用户运行
- 使用 dumb-init 处理信号
- 添加健康检查

## 目标镜像大小
- 目标：小于800MB
- 当前：1.53GB
- 预期减少：约50%

# MoonTV 永久 Dev 版本声明

> **文档版本**: dev (永久开发版本) | **更新日期**: 2025-10-14 | **状态**: 永久有效

## 🎯 永久 Dev 版本哲学

### 核心宣言

MoonTV 项目正式宣布 **dev 作为永久开发版本标识**，这是一个经过深思熟虑的战略决策，旨在简化版本管理、专注功能开发、提升团队效率。

**版本哲学宣言**:

> "我们选择 dev 作为永久的开发版本标识，不再受传统版本号束缚。我们相信，优秀的软件应该通过功能和质量来体现价值，而不是通过复杂的版本号。dev 代表着持续进化、无限可能的开发精神。"

### 决策背景

```yaml
传统版本管理的困境:
  - 版本号疲劳: 频繁的版本号更新造成认知负担
  - 发布压力: 版本号驱动的发布节奏带来不必要的压力
  - 沟通成本: 复杂的版本号增加团队沟通成本
  - 维护复杂: 多版本并行维护带来管理复杂度

MoonTV 的特殊定位:
  - Docker 专用版本: 专注于 Docker 镜像制作
  - 企业级部署: 面向企业级用户和部署场景
  - 持续迭代: 功能驱动的持续开发模式
  - 质量优先: 以质量而非版本号衡量价值
```

## 📊 版本体系重新定义

### 四版本体系 (永久架构)

```yaml
1. 开发版本: dev (永久)
   - 定位: 永久的开发环境标识
   - 特点: 从不改变，统一标识
   - 用途: 所有开发活动的统一标识
   - 更新: 通过 Git 提交体现变化

2. 应用版本: v3.2.0 (同步)
   - 定位: 与上游仓库同步的软件版本
   - 特点: 严格与上游保持一致
   - 用途: 软件功能版本标识和更新检查
   - 更新: 上游仓库更新时同步更新

3. 生产版本: v6.0.0 (按需)
   - 定位: 稳定功能的正式发布版本
   - 特点: 基于稳定的开发里程碑创建
   - 用途: 生产环境部署和长期维护
   - 更新: 重要里程碑时按需创建

4. 测试镜像: moontv:test (持续)
   - 定位: 生产就绪的测试镜像
   - 特点: 自动化构建和测试验证
   - 用途: 功能验证和测试部署
   - 更新: CI/CD 自动化持续更新
```

### 📝 仓库管理策略

#### 版本控制策略
- 使用 `main` 分支作为主要开发分支
- 使用 `master` 分支作为稳定发布分支
- 采用语义化版本号 (Semantic Versioning)
- 重要版本创建 Git 标签

#### 发布流程
1. 本地开发和测试
2. 代码质量检查 (ESLint, TypeScript)
3. 安全审查和修复
4. 创建提交和本地测试
5. 推送到GitHub仓库
6. 创建Release标签

#### 分支保护
- 重要分支需要代码审查
- 安全修复优先级最高
- 定期清理无用分支和标签
